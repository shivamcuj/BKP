State Space Search

The fundamental process in AI for solving problems by exploring possible configurations. From playing Chess to finding the best route on Google Maps.

What is State Space Search?

State Space Search is the process of finding a solution to a problem by exploring a set of possible configurations (called states). This is how machines tackle problems like humans do in gaming scenarios (Tic-Tac-Toe, Chess).

Graph Representation

The problem is organized as a graph or tree where:

  • Nodes: Represent different states (configurations).
  • Edges: Represent transitions (moves) between states.

The Formal Definition

A problem is defined by the set S: {S, A, Action, Transition, Cost}.

It includes the Initial State, Goal State, and the Set of Actions ($A$).

Key Terminology

State

A snapshot of the system at a specific moment. It represents exactly what the problem looks like right now.

Start & Goal State

Start State: Where you begin (Initial configuration).
Goal State: Where you want to reach (The solution).

Operators / Actions

The steps you can take to move from one state to another. In navigation, this could be "Turn Left" or "Walk".

Path & Search Strategy

Path: The sequence of steps from Start to Goal.
Search Strategy: The method used to decide which path to explore first.

Classic Example: The 8-Puzzle

The 8-Puzzle consists of a 3x3 board with eight numbered tiles and one Blank Space. A tile adjacent to the blank space can slide into it.

Goal Arrange the mixed-up tiles into a specific order (e.g., 1-8).
Actions (Operators) Move the blank space: UP, DOWN, LEFT, RIGHT.
Path Cost Each step costs 1. Total cost = Number of steps.

Real World Examples

Route Finding (Google Maps)

Imagine navigating from your Hostel to the Exam Hall.

  • Start: Current Location (Hostel).
  • Goal: Destination (Library/Hall).
  • Actions: Turn Left/Right, take Bus/Metro.
  • Search: Google Maps calculating the fastest route.

The Water Jug Problem

A classic AI logic puzzle often used to demonstrate state spaces.

  • Goal: Measure a specific amount of water using 2 jugs.
  • Constraint: Jugs have no measurement markings.
  • Actions: Fill jug, Empty jug, Pour one into another.
